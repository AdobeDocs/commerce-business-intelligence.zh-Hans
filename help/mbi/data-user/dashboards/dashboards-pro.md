# 现成功能板

[!DNL MBI] 包括现成功能板，可提供对您业务的分析。 借助功能板，您可以检查基本量度的运行状况，例如用户生命周期收入、重复购买次数、在指定时间段内购买的热门产品等。 这些预配置功能板旨在帮助您做出明智的业务决策。

>[!NOTE]
>
>对这些功能板的访问权限取决于您的帐户类型和访问级别。 如果您看不到这些功能板，请联系 [支持](../../guide-overview.md).

## 报告可用性

对于 `Customers` 和 `Executive Summary` 功能板，某些报表仅根据您商店的结帐配置可用。 具体来说，如果您的商店允许来宾结账，或者不允许来宾结账。

## 客户（允许来宾结账）

“客户”（允许来宾结账）功能板提供有关客户群的信息，如他们的购买行为。 此功能板可帮助您提高客户维系率并确定哪些客户带来了最高收入。

### 报表

| 名称 | 描述 |
|---|---|
| `Orders by New Customers (Past 30 Days)` | 过去30天内从未下过订单的客户的订单。 |
| `Orders by Existing Customers (Past 30 Days)` | 过去30天内从以前至少下了一次订单的客户那里下达的订单。 |
| `Total Unique Customers (Past 30 Days)` | 过去30天内下单的独特客户计数。 |
| `Orders by New vs Existing Customers` | 没有先前订单的客户与至少有一次先前订单的客户的订单数。 |
| `Subsequent Order Probability (All Time)` | 下订单的客户再次下订单的可能性。 |
| `% of Customers with Multiple Orders (All Time)` | 已下达超过一次订购的所有客户的百分比。 |
| `Median Time Between Orders (All Time)` | 每位客户下一次订单之间间隔的中间时间。 |
| `Subsequent Order Probability` | 下订单的客户再次下订单的可能性（按订单号划分）（即，拥有一次订单的客户中，有一次下单的客户占百分比，有两次下单的客户占百分比，有三分之一，依此类推）。 |
| `Time Between Orders` | 客户在订单之间的平均时间和中间时间，按订单编号划分（即订单一和订单二、订单二和订单三之间的时间，依此类推）。 |
| `Number of Customers - Lifetime Orders` | 对于在客户生命周期中下达的给定数量的订单，已下达该数量的订单的客户数量以及该数量代表的整个客户基础的百分比。 |
| `One-Time Customers who Bought 3-6 Months Ago` | 在三到六个月前首次购买且仅购买的客户。 |
| `Avg LTV by First Order` | 比较跨同类群组的累积平均客户生命周期收入。 同类群组由客户首次购买的月份定义。 例如， `Jan 2020` 同类群组显示首次购买是在2020年1月的客户的累积平均LTV。 |
| `Customer's First 30 Day vs Lifetime Revenue` | 比较客户首次购买后30天内的平均收入与其整个生命周期内的平均收入。 每个泡泡都对应一个运输区域，每个泡泡的大小代表从该区域获得的客户数量。 |

## 客户（不允许来宾结账）

“客户”（不允许客人结账）功能板提供有关客户群的信息，例如他们的购买行为以及从帐户注册到订单投放的转化。 此功能板可帮助您提高客户维系率并确定哪些客户带来了最高收入。

### 报表

| 名称 | 描述 |
|---|---|
| 帐户注册（过去30天） | 过去30天内在您的商店中注册帐户的人数。 |
| 已注册（过去30天）的帐户，订购1个或更多 | 过去30天内在您的商店注册帐户，并且还下了至少一次订单的人数。 |
| 从注册到订购的转化百分比（过去30天） | 过去30天中已下订单的帐户百分比。 |
| 从注册到订购的转化百分比 | 已下订单的已注册帐户的百分比，按注册月份计算。 |
| 按新客户和现有客户划分的订单数 | 没有先前订单的客户与至少有一次先前订单的客户的订单数。 |
| 后续顺序概率（全时） | 下订单的客户再次下订单的可能性。 |
| 拥有多个订单的客户（全时）百分比 | 已下达超过一次订购的所有客户的百分比。 |
| 订单间隔的中间时间（所有时间） | 每位客户下一次订单之间间隔的中间时间。 |
| 后续顺序概率 | 下订单的客户再次下订单的可能性（按订单号划分）（即，拥有一次订单的客户中，有一次下单的客户占百分比，有两次下单的客户占百分比，有三分之一，依此类推）。 |
| 订单间隔时间 | 客户在订单之间的平均时间和中间时间，按订单编号划分（即订单一和订单二、订单二和订单三之间的时间，依此类推）。 |
| 客户数量 — 存留期订单 | 对于在客户生命周期中下达的给定数量的订单，已下达该数量的订单的客户数量以及该数量代表的整个客户基础的百分比。 |
| 3-6个月前购买的一次性客户 | 在三到六个月前首次购买且仅购买的客户。 |
| 按首次订购的平均LTV | 比较跨同类群组的累积平均客户生命周期收入。 同类群组由客户首次购买的月份定义。 例如，2020年1月的同类群组显示首次购买是在2020年1月的客户的累积平均LTV。 |
| 客户前30天与生命周期收入 | 比较客户首次购买后30天内的平均收入与其整个生命周期内的平均收入。 每个泡泡都对应一个运输区域，每个泡泡的大小代表从该区域获得的客户数量。 |

## 执行摘要（允许来宾结账）

“执行摘要”（允许客人结账）功能板为您提供了业务在订单和收入方面如何运作的简洁图片。 此功能板旨在让高管全面了解业务绩效，但也能让其他人洞察到。

### 报表

| 名称 | 描述 |
|---|---|
| 收入（当月） | 您的商店在当月产生的收入。 在这种情况下，收入被定义为客户在订单上支付的最终价格。 |
| 收入（过去6个月，按日） | 每日总收入，与前七天的平均每日收入重叠。 在这种情况下，收入被定义为客户在订单上支付的最终价格。 |
| 收入变动(MoM MTD)% | 当月（迄今为止）与上个月相同部分的收入比较。 |
| 来自新客户与现有客户的收入（当月） | 当月（到目前为止）的收入归属于新（首次）客户与现有客户（下次或以后次订购）的收入。 |
| 平均订单值（当月） | 当月（迄今为止）下单的每日平均值。 订单值定义为客户在订单上支付的最终价格。 |
| 订单（当月） | 当月（到目前为止）在您的商店内下订单的数量。 |
| 订单更改百分比(MoM MTD) | 当月（迄今为止）订单数与上个月相同部分的比较。 |
| 按新客户划分的订单（当月） | 当月来自以前从未下过订单的客户的订单。 |
| 按现有客户（当月）划分的订单 | 当月来自以前至少下过一次订单的客户的订单。 |
| 按新客户与现有客户的订单（按周列出的当前年份） | 没有先前订单的客户与至少有一个先前订单的客户的订单数（截至目前），对应当年的每周。 |

## 执行摘要（不允许来宾结账）

“执行摘要”（不允许客人结账）功能板为您提供了业务在订单、收入和帐户注册方面的简洁信息。 此功能板旨在让高管全面了解业务绩效，但也能让其他人洞察到。

### 报表

| 名称 | 描述 |
|---|---|
| 收入（当月） | 您的商店本月产生的收入。 在这种情况下，收入被定义为客户在订单上支付的最终价格。 |
| 收入（过去6个月，按日） | 每日总收入，与前七天的平均每日收入重叠。 在这种情况下，收入被定义为客户在订单上支付的最终价格。 |
| 收入变动(MoM MTD)% | 将本月迄今的收入与上个月同期的收入进行比较。 |
| 来自新客户与现有客户的收入（当月） | 当月（到目前为止）的收入归属于新（首次）客户与现有客户（下次或以后次订购）的收入。 |
| 平均订单值（当月） | 当月（迄今为止）下单的每日平均值。 订单值定义为客户在订单上支付的最终价格。 |
| 订单（当月） | 当月（到目前为止）在您的商店内下订单的数量。 |
| 订单更改百分比(MoM MTD) | 当月（迄今为止）订单数与上个月相同部分的比较。 |
| 帐户注册（当月） | 本月迄今新注册的账户数。 |
| 从注册到订购（当月）的转化百分比 | 本月迄今为止已下订单的帐户所登记的百分比。 |
| 从注册到订购的转化百分比（按周列出的当前年份） | 今年到目前为止已下单的每周注册帐户的百分比。 |

## 订单数

“订单”功能板提供有关订单交易量、其状态、使用的优惠券代码、生成的收入以及使用的付款方法的分析。 例如，您可以跟踪履行过程，并确保不存在问题或出现瓶颈。

>[!NOTE]
>
>此功能板上的报表适用于连接到具有任一类型配置（来宾结账、无来宾结账）的商店的帐户。

### 报表

| 名称 | 描述 |
|---|---|
| 订单数（过去30天） | 过去30天内在您的商店下单的数量。 |
| 收入（过去30天） | 过去30天内您的商店产生的收入。 收入定义为客户在订单上支付的最终价格。 |
| 平均订单值（过去30天） | 过去30天的订单平均值。 订单值定义为客户在订单上支付的最终价格。 |
| 订单数 | 每月在您的商店中下达的订单数。 |
| 按付款方式收入 | 您的商店产生的收入，按付款方式划分。 收入定义为客户在订单上支付的最终价格。 |
| 新客户与现有客户的AOV | 在您的商店中下达的订单的月平均值，按没有先前订单的客户与至少具有先前一次订单的客户所下达的订单进行细分。 订单值定义为客户在订单上支付的最终价格。 |
| 按状态划分的订单百分比（过去30天） | 过去30天中每天处于每种订单状态的订单的百分比。 |
| 未完成订单（在1天前创建） | 一个列表，其中列出了一天前下达的仍处于未完成状态（未取消或完成）的所有订单。 |
| 每小时订单数（过去7天） | 按小时订购数量。 |
| 收入详细信息（过去30天） | 过去30天的每日收入划分到总收入值的所有组成部分。 |
| 按优惠券代码列出的订单详细信息（过去30天） | 对于您商店提供的每个优惠券代码，详细说明了该优惠券代码的使用方式以及在过去30天中它带来的回报。 |
| 具有优惠券的订单百分比（过去30天） | 过去30天内使用优惠券的订单与未使用优惠券的订单的百分比。 |

## 产品

“产品”功能板按订购产品、其总商品价值(GMV)以及购买和退货的热门产品显示总体产品性能。 它有助于您在购买和退货之间取得平衡，并确定产品的成功与受欢迎程度。 您的商店必须 [已配置以跟踪退款](https://experienceleague.adobe.com/docs/commerce-admin/customers/customer-accounts/store-credit/credit-configure.html) 来填充这些图表。

>[!NOTE]
>
>此功能板上的报表适用于连接到具有任一类型配置（来宾结账、无来宾结账）的商店的帐户。

### 报表

| 名称 | 描述 |
|---|---|
| GMV（过去30天） | 过去30天内销售的所有产品的商品总价值。 GMV定义为订购数量乘以每个产品的基本价格。 |
| 已退回的GMV（过去30天）百分比 | 过去30天内购买产品的GMV百分比，这会导致退款。 |
| 订购的产品数量（过去30天） | 过去30天内订购的物料总数。 |
| 已购产品（过去30天）已退回的百分比 | 过去30天内导致退款的购买项目百分比。 |
| 总商品价值 | 按月销售的所有产品的商品总价值。 GMV定义为订购数量乘以每个产品的基本价格。 |
| 每个产品的购买次数与退款率（过去30天） | 对于每种产品，将过去30天订购的总数量与退货率进行比较。 每个泡沫的规模代表退款率。 |
| 产品性能详细信息（过去30天） | 过去30天内按产品SKU和产品名称列出的销售和后续退款的详细信息。 |
| GMV购买的最多产品（过去30天） | 过去30天内销售的产品带来的收入最多（前10天）。 |
| 按GMV退还的最高产品（过去30天） | 过去30天内购买的产品因退款而损失的GMV最多（前10天）。 |
| 按数量列出的最常购产品（过去30天） | 过去30天内销售的产品数量最多（前10天）。 |
| 按数量（过去30天）列出的最高退货产品 | 过去30天内购买的产品，可获得最大退货量（前10天）。 |
